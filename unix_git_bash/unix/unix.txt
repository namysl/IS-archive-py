/etc/fstab		-> file system that is automatically mounted to
			   linux when booting

sysfs			-> virtual file system, outputs the information of
			   devices and drivers from kernel space to user
			   space and can be used to make settings for 
			   devices and drivers

udev			-> user space tool, uses the information provided
			   by sysfs to implement all the functions of devfs
			   (devfs runs in kernel space)

tmpfs			-> can use your RAM (like a virtual disk) and swap
			   partition for storage
			   it's a file system, not a block device

SUID			-> set-user identification, chmod 4555 "path to file"
			   non-root user can change his own password by this

SGID			-> set-group identification, chmod 2555

SVTX (sticky bit)	-> used on shared directories, user cannot deleted
			   other user's file, even if he can rwx it
			   chmod +t "path to dir" or chmod 1777 "path"
